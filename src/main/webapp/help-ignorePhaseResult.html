Define how to decide the status of the whole MultiJob Project, based on the status of current phase.

Never Usual behaviour: if something goes wrong in this phase, MultiJob Project will have Failed status.
Stable or Unstable but not Failed If status of this phase is Unstable the MultiJob Project will have Success status still.
Always The same as previous mode, but Failed status will be ignored for MultiJob Project status too.

Supported command for DSL-script: ignorePhaseResult( 'ALWAYS' or 'NEVER' or 'STABLE' )
